Get the icon.

IconForge decides what object to draw, writes the image prompt, runs it through the agy CLI, then cuts the result to the right shape for the Mac you are on and builds every size Apple asks for. What comes out is an icns you can drop into a bundle, not a square PNG you still have to deal with.

Preview on both desktops, plus 32pt and 16pt, because that is where icons usually fall apart.

From prompt to icns

  1. You describe the appA name and one line about what it does. Colour, style and finish are optional.
  2. It picks a subjectLeave the subject blank and the model names the object worth drawing, mixing the literal choice with a lateral one. The answer lands back in the field, so you can edit it and try again.
  3. agy draws itThe prompt describes a 3D render of one object on a gradient, never an app icon, because asking for the artifact is what makes models draw borders and badges.
  4. The shape goes onSuperellipse corners and a soft shadow, or a plain square when macOS is doing the rounding itself. The finish adds a lit top edge and a shaded base.
  5. Everything gets writtenTen iconset sizes, the icns from iconutil, a Windows ico, the finished 1024, and the untouched original.

Details that matter

Four at once, keep one

Ask for up to four icons and each gets its own subject, material and camera angle. They arrive as a row under the preview; click the one you want and it becomes the icon everything else acts on.

Edit it in words

"Make the bird bigger." The existing artwork goes back to the model with orders to keep the subject, composition and palette and change only that. The original stays where it was.

Finishes that cost nothing

A lit top lip, a shaded base, a glossy dome, a deeper shadow, richer colour. These are local Core Graphics passes over the stored artwork, so trying all five takes milliseconds and no model calls.

It knows about the plate

macOS 26 wraps a legacy icns in its own rounded plate, which turns your careful margin into a white border. Full bleed hands the system a plain square and lets it do the rounding.

Rerolls change their mind

A subject IconForge chose gets re-derived on the next press, avoiding the last few it used for that app, so you get a different idea rather than the same object drawn again.

Hand it to an agent

One button copies an instruction naming the exact files and telling a coding agent to install the icon on whatever you have open, rebuild and reinstall.

Nothing gets overwritten

Every run lands in its own timestamped folder with the prompt that produced it, so you can go back and see exactly what you asked for.

It tells you what went wrong

A missing agy binary, a model name agy does not know, an absent iconutil. Each one gets a message that says what to do about it instead of a silent failure.

Native, and small

A SwiftUI app built as a plain Swift Package. No Xcode project, no dependencies, no bundled runtime.

Install

# builds a release bundle and copies it to /Applications
./install.sh

# or run it straight out of the build folder
./build_app.sh && open build/IconForge.app

Needs macOS 14 or later, the Xcode command line tools, and agy somewhere your shell can find it. If the app cannot see agy, paste the output of which agy into Settings.
